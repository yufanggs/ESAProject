Modular Architecture

Identifying the subsystem

This shows how we group the game elements into different subsystems.

In this case, there will be five subsystems which are movement, location, timing, cards and scoring subsystems.

Interactions between subsystems

This shows the flow and interactions between individual subsystems.

When the user start playing by throwing a dice, the massage will be passed to the movement subsystem. The number on the dice translates into the number of movements the charactor piece can move. When the charactor lands on a location in the game board, the massage will be passed to the location subsystem. Depending on the type of location the charactor lands on, there will be different outcomes. This outcomes will the passed to the user to perform different actions. If it lands on landmarks or special MRT, the user will draw question card or oppotunity cards respectively. And this will be happening in the card subsystem. If the user draws a question cards, the massage will be passed to timing subsystem and scoring subsystem at the same time. In timing subsystem, the timer will start. If the user manages to give a response within time limit, the massage will be passed to the scoring system. If the user fails to give a response within the time limit, there is no scores straightaway. In the scoring system, if the response gived by the user matched the question answer, the user will be awarded one point, and he will exit the system. His round has finished.
